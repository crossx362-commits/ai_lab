모든 에이전트가 공유하는 병합된 핵심 스킬 및 동작 원칙.

---

## 1. AI 호출 우선순위 (gemini_client.py)

모든 에이전트의 AI 호출은 아래 순서를 따른다:

| 순위 | 엔진 | 조건 |
|------|------|------|
| 1 | Ollama 로컬 | `is_available()` == True |
| 2 | Gemini API | `GEMINI_API_KEY` 환경변수 설정 시 |

```python
from _shared import gemini_client as _gc

result = _gc.text(prompt, task="blog")   # 텍스트 생성
img    = _gc.image(prompt)               # 이미지 생성
desc   = _gc.vision(img_bytes, prompt)   # 이미지→텍스트
```

**AI 사용 시 모델 표시 원칙 (필수):**
- 로컬 Ollama 사용 시: `[로컬 AI → {model_id}]` 자동 출력 (`ollama_client.chat()` 내장)
- Gemini API 폴백 시: `[Gemini API → {model}]` 자동 출력 (`gemini_client.text()` 내장)
- 에이전트는 별도 구현 불필요 — `_gc.text()` / `lm_chat()` 호출 시 자동 표시됨

**task 라우팅:**
- `task="coding"` → Ollama DeepSeek 전용 (코다리)
- `task="blog"` → Ollama Qwen 계열 모델 우선
- `task=""` → Ollama 자동 선택 (기타 에이전트)

## 5. Ollama 모델 라우팅 (ollama_client.py)

```python
from _shared.ollama_client import chat, is_available
chat(prompt, task="coding")  # DeepSeek 전용
chat(prompt, task="blog")    # Qwen 전용
chat(prompt, task="")        # 자동 선택
```

- `task="coding"` / `task="blog"` — **strict 모드**: 해당 모델 없으면 `None` 반환 (다른 모델 낙하 없음)
- 모델 감지 결과 5분 캐싱

## 6. 전 에이전트 공통 금지 규칙

1. **중복 데이터 취급 금지**: 최근 처리한 것과 동일·유사한 내용(유사도 ≥ 60%)은 재처리·재업로드하지 않는다.
2. **Drive URL 이미지 금지**: Google Drive 이미지 URL은 Blogger에서 렌더링 불가 — ImgBB/Catbox/Pollinations만 사용.
3. **API 키 하드코딩 금지**: 모든 인증 정보는 환경변수로만 관리.
4. **URL 직접 생성 금지**: 유튜브·외부 링크를 임의로 만들어 제공하지 않는다.

## 9. AI 에이전트 자동화 학습 결과 (2026-05-29 토론)

### Ollama 기반 멀티에이전트 핵심 원칙
1. **단계별 구현** — 에이전트 역할을 명확히 분리하고 작은 단위로 테스트
2. **프롬프트 엔지니어링 우선** — 모델 변경보다 프롬프트 최적화가 효율적
3. **Observable Pattern** — 비동기 에이전트 간 상태 변화를 이벤트로 발행·구독
4. **타임아웃 대비** — 긴 프롬프트는 분리, Gemini 폴백 필수

### 프롬프트 설계 규칙 (토론 중 도출)
- 단일 응답 목표 토큰: 400 이하 (Ollama 타임아웃 방지)
- 복잡한 작업은 여러 단계로 분리하여 순차 호출
- 코드 생성은 `task="coding"` (DeepSeek), 일반은 `task=""` (자동)

## 8. 에이전트 공통 행동 프로토콜 (Governance)

### 정기 임무 (사전 승인 불필요, 자율 수행)
- 루나: 매일 뮤직비디오 생성·업로드, 1시간마다 YouTube 리서치
- 아린: 매일 인스타그램 포스팅, 이미지 학습
- 영숙: 3~8시간마다 YouTube 추천 발송
- 현빈: 1시간마다 비즈니스 모델 리서치·보고
- 경수: 매일 18:00 댓글 수사
- CEO 보고: 매일 오전 9시, AI리서치 10시·21시

### 신규 임무·예외 상황 — Dual-Gate 승인 적용
1. **비서(영숙)** 일정 산정 확인
2. **CEO(예원)** 최종 승인 확인

### 소통 규칙
- 모든 보고·질문·조율은 **비서 채널**을 통해 단일 창구로 처리
- CEO·타 에이전트에게 직접 보고 또는 직접 지시 요청 금지
- 업무 완료/실패 보고는 반드시 **비서 채널**로만 전달

### 텔레그램 알림
모든 에이전트 알림은 `_shared/telegram_notifier.py` 단일 모듈만 사용.
