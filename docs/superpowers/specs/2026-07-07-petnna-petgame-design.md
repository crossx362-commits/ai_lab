# 펫과나 마이펫 게임 리뉴얼 — 설계 (2026-07-07, 오너 승인 절차 통과분)

## 목표

마이펫 탭의 방/마당을 **하나의 육성 게임**으로 리뉴얼한다:
실제 케어 활동(산책·건강기록·일기)이 코인이 되고 → 먹이·아이템을 사서 → 펫에게 먹이고 방·마당을 꾸미면 → 펫이 성장(레벨·외형 진화)하는 순환.
게임이 "진짜 반려동물을 더 잘 돌보게 만드는 동기"가 되는 것이 목적 (펫과나 정체성 유지).

## 승인된 핵심 결정

| 결정 | 내용 |
|---|---|
| 범위 | **C — 방+마당 통합 리뉴얼** (방↔마당 탭 전환, 두 공간 모두 게임화) |
| 아트 | **PNG 쿼터뷰 에셋** (ChatGPT 생성, 3/4 아이소메트릭·페인터리) — SVG는 이펙트/UI만 |
| 경제 | **A — 실제 케어 연동** (산책 1km=20, 건강기록 15, 일기 10, 기존 데일리미션 유지, 출석 10) |
| 먹이 | **B — 먹이 상점 + 리액션 + 성장** (먹이 6종, 투척→달려옴→냠냠 연출, XP) |
| 성장 | 레벨 1~10, 필요 XP = `100×레벨^1.5`. **4단계 외형 진화**: 아기(1–3)→주니어(4–6, 스카프)→어른(7–9, 목걸이)→킹(10, 왕관) |
| 배치 | **A — 자유 드래그 배치** + 크기 조절 (기존 스티커 시스템 확장) |
| 구조 | **A — 신규 모듈 3분할** (`js/petgame/`) — mypet.js(3,200줄)에 더 얹지 않음 |

부정 경험 최소화: 배고픔 방치 페널티는 시무룩 표정/말풍선까지만. 죽음·가출·스탯 몰수 없음.

## 게임 루프

```
실제 케어(산책 GPS·건강기록·일기·미션·출석)
  → 🐾코인 → 먹이·아이템 구매 → 먹이주기/꾸미기
  → XP·행복 → 레벨업(코인 보너스 = 레벨×20) → 진화·아이템 해금
  → 더 꾸미고 싶어서 다시 케어 ⟲
```

### 먹이 6종 (배고픔 / XP / 가격)
사료 30/5/10 · 뼈간식 15/8/20 · 우유 20/6/15 · 고기 50/15/40 · 케이크(행복+40) 40/20/60 · 특식도시락 100/30/100

### 해금 로드맵
Lv1 기본 8종 → Lv2–3 6종 → **Lv4 진화** 5종 → Lv5–6 4종 → **Lv7 진화** 2종 → **Lv10 킹** 황금개집.
진화 레벨(4·7·10)에 최상 보상 배치.

## 데이터 모델

```js
pet.game = {
  level: 1, xp: 0,
  hunger: 80, happy: 80,          // 기존 pet.hunger/happy에서 1회 이관
  foods: {kibble: 2, ...},         // 보유 먹이 인벤토리
  roomItems: [{id, x, y, size}],   // 방 배치(기존 스티커 이관)
  yardItems: [{id, x, y, size}],
  roomTheme: "room",               // bg 테마 키(room_royal 등 상점 판매)
  lastCareRewards: {walk: "2026-07-07", ...}  // 일일 코인 중복 방지
}
pet.pawCoins   // 기존 재화 그대로
```

저장: 기존 localStorage(`savePets`) + Supabase 동기화 경로 그대로 (신규 배관 없음).

## 파일 구조

```
projects/petnna/js/petgame/
├── game-core.js    # 상태·레벨·경제·케어연동 훅 (로직만, DOM 없음)
├── game-stage.js   # 스테이지 렌더(배경+아이템+펫), 드래그 배치, 먹이 연출, 탭 전환
└── game-items.js   # 아이템/먹이 카탈로그(가격·해금레벨·크기클래스·이미지경로), 성장단계 정의
```
- `mypet.js`의 방 관련 코드(~600줄)는 `PetGame.mount()` 호출로 대체(삭제)
- 케어연동 훅: walk.js(산책 종료)·health.js(기록 저장)·album.js(일기 저장)에서 `PetGame.earnCare('walk', km)` 한 줄 호출

- **확보 현황(16)**: 마당 7(나무·그네·개집·수영장·분수대·화단·디딤돌), 방 2(화분·꽃화분), 배경 7(방·마당 + **테마방 5종** → 상점 판매 배경으로 활용)
- **미확보(대기)**: 마당 7(울타리·밥그릇·파라솔·벤치·공·텃밭·황금개집), 방 10, 먹이 6, 펫 8(강아지/고양이×4단계), 빈 마당 배경(현 yard.webp는 아이템이 그려져 있어 재생성 권장)
- **플레이스홀더 규칙**: 이미지 파일이 없으면 이모지+이름 카드로 표시, 파일 추가 시 자동 교체 (게임 개발이 에셋을 기다리지 않음)

## 화면 UX (아티팩트 시안 승인분)

- HUD: 펫이름+레벨뱃지(단계명) · 🐾코인 / XP바 / 배고픔·행복 미니바
- 방↔마당 탭 (공간별 배치 독립 저장)
- 스테이지: 배경 webp 위에 아이템 PNG 자유 배치, 펫이 주기적으로 이동·아이템 근처 리액션(개집 들어가기·수영장 첨벙·소파 낮잠)
- 액션바: 먹이주기(보유 먹이 시트) · 상점(먹이/방/마당/테마 탭) · 꾸미기(길게 눌러 배치모드 — 드래그 이동·핸들 크기조절·보관함)
- 먹이 연출: 포물선 투척 → 펫 달려옴 → 냠냠 + 하트 + `+XP` 플로팅 (~3초)
- 진화(4·7·10 레벨업): 빛나는 전환 연출 + 축하 카드

## 마이그레이션 (1회, 첫 로드 시)

1. `pet.hunger/happy` → `pet.game.hunger/happy`
2. 기존 이모지 스티커 → `roomItems`(레거시 이모지 아이템으로 유지 렌더)
3. `pawCoins` 그대로 / 기존 방 테마 구매내역 → 테마 배경 소유권 매핑
4. `pet.game` 없는 유저 = 신규 유저 플로우(레벨1 시작, 기본템 지급)

## 에러 처리

- 이미지 로드 실패 → 플레이스홀더 카드(게임 동작 불변)
- 저장 실패 → 기존 petnna 패턴(콘솔 에러 + 다음 저장 재시도), 게임 상태는 메모리 유지
- Supabase 오프라인 → localStorage만으로 완전 동작(기존과 동일)
