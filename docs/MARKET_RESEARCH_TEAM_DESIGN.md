# 시장조사 에이전트 팀 — 설계안 (v0.1)

> 작성 2026-06-27 · 상태: **설계 검토용 (코드 미구현)**
> 목적: 주가에 영향을 주는 이슈(공시·거시·뉴스·심리)를 지역별 전담 에이전트가 수집·정규화해, 소미(국내주식 판단)와 예원(의사결정)에 공급한다.

---

## 1. 배경과 방향

- 현재 소미는 KIS API의 **수급 데이터(가격·거래량·대차·공매도)** 만 본다. 시장을 움직이는 외부 이슈(공시, 환율, 미국장, 뉴스)는 보지 못한다.
- 이슈 수집을 소미에 얹으면 소미가 비대해지고 책임이 흐려진다. → **별도의 지역별 조사 팀**으로 분리하고, 소미는 그 결과를 "입력"으로만 받는다.
- 시장은 시간대별로 움직인다(미국장 새벽, 아시아 오전, 유럽 오후). 지역별로 나누면 수집 시점·소스·관점을 각자 최적화할 수 있다.

---

## 2. 팀 구성

기존 3명(예원·영숙·소미)에 **조사관 3명 + 통합 데스크 1**을 추가한다. 이름은 가칭.

| 에이전트 | 담당 시장 | 핵심 책임 |
|---|---|---|
| 🇺🇸 미국 조사관 (가칭 "행크") | 미국 | S&P500·나스닥, 연준/FOMC·금리·CPI, 빅테크·반도체 동향, USD 흐름 |
| 🌏 아시아 조사관 (가칭 "유나") | 한국·일본·중국 | KOSPI/KOSDAQ, **DART 공시**, 한국은행 금리·환율, 닛케이·항셍, 외국인 수급 배경 |
| 🇪🇺 유럽 조사관 (가칭 "레온") | 유럽 | 유로스톡스·DAX, ECB 정책, 에너지·원자재 |
| 🧭 시장 데스크 (예원이 겸임 or 신설) | 통합 | 3개 지역 브리프를 받아 **하나의 시장 브리프**로 종합, 한국 종목 영향도 태깅 |

> 국내주식이 최종 타깃이므로 **아시아(특히 한국)팀이 1순위**, 미국팀이 2순위(거시 파급), 유럽팀이 3순위다.

## 3. 각 조사관의 수집 대상 · 소스

| 분류 | 수집 항목 | 소스(후보) | 키 필요 |
|---|---|---|---|
| **공시** | watchlist 종목·관련주 공시(증자·CB/BW·실적·대량보유·단일판매계약) | **DART OpenAPI** | 무료 키(미보유, 발급 필요) |
| **거시(한국)** | 기준금리·환율·국채금리 | 한국은행 **ECOS** API | 무료 키(발급 필요) |
| **거시(미국)** | 연준 금리·CPI·고용·국채 | **FRED** API | 무료 키(발급 필요) |
| **지수/해외증시** | S&P·나스닥·닛케이·항셍·DAX | KIS 해외지수 / Finnhub·AlphaVantage 무료티어 | 일부 무료 키 |
| **일정(이벤트)** | FOMC·CPI·옵션만기·실적발표 캘린더 | 공개 경제캘린더 RSS/스크래핑 | 불필요(또는 무료) |
| **뉴스** | 종목·산업 헤드라인 | 네이버 금융 RSS(국내), 글로벌 RSS/NewsAPI | 국내 RSS 무료 / 글로벌 일부 유료 |
| **심리** | 공포탐욕지수·VIX | CNN F&G(비공식), VIX 인용 | 불필요 |
| **요약·영향도** | 수집 텍스트 → 요약·분류·한국증시 영향도 | **Gemini/GPT/Claude/Ollama (.env 보유)** | 보유 ✅ |

**키 현황 요약**: LLM·KIS·UPBIT 키는 보유. **DART·ECOS·FRED 등 데이터 소스 키는 신규 발급 필요**(대부분 무료). 사용자가 보유했다고 한 키가 있으면 `.env`에 추가 후 매핑.

## 4. 데이터 흐름

```
[지역 조사관] 수집(API/RSS) ──▶ 원문 정규화 ──▶ LLM 요약·영향도 평가
        │                                              │
        ▼                                              ▼
 output/research/region_<us|asia|eu>.json    (종목 태그 + 영향도 ±)
        │
        ▼
[시장 데스크] 3개 지역 종합 ──▶ output/research/market_brief.json + .md
        │
        ├─▶ 영숙 아침 브리핑에 "오늘의 시장 이슈" 포함
        ├─▶ 소미: 종목 공시 + 시장 브리프를 보고/판단에 반영
        └─▶ 예원: 의사결정 컨텍스트
```

- 각 조사관은 **구조화 JSON**을 남긴다(헤드라인, 출처 URL, 시각, 관련 종목코드, 영향도 −2~+2, 한 줄 요약).
- 시장 데스크는 종목코드 기준으로 묶어 **종목별 이슈 카드**를 만들고, 소미가 이를 그대로 인용한다.

## 5. 아키텍처 통합 (기존 패턴 준수)

- **폴더**: `projects/ai-team/skills/미국_조사관/tools/`, `아시아_조사관/tools/`, `유럽_조사관/tools/` (또는 단일 `시장조사/` 스킬 + 지역 모듈). 각 폴더에 `SKILL.md`.
- **공통 모듈**: `_shared/env`(키)·`_shared/llm`(요약)·`_shared/notify`(텔레그램)·`_shared/process`(락) 재사용.
- **실행**: 데이터 수집은 단발 스크립트 → `schedule_manager`(실행기)가 cron으로 호출, 또는 상시 데몬 불필요(하루 수 회면 충분).
- **등록**: `agent_controller.py`(실행 대상) + `src/agents.ts`(대시보드 카드·색상) + `_shared/notify.py`(현황 라벨).
- **대시보드**: 신규 3명을 `AGENT_MAP`/`agents.ts`에 추가하면 가상사무실·아티팩트 대시보드에 자동 노출.

## 7. 단계별 로드맵

1. **1단계 (한국 코어)**: DART 공시 수집기 + 한국은행 환율/금리 + KIS 지수 → 소미 보고에 "관련 공시·거시" 섹션. *가장 직접적·고신뢰.*
2. **2단계 (미국 거시)**: FRED + 미국 지수 + FOMC/CPI 캘린더 → 아침 브리프에 "미국장·이벤트".
3. **3단계 (유럽·뉴스·심리)**: 유럽 지수 + 뉴스 RSS(LLM 필터) + 공포탐욕/VIX.
4. **4단계 (영향도 → 점수)**: LLM이 이슈를 종목별 영향도(±)로 평가, 소미 점수에 가중(예: 악재 공시 −, 호재 계약 +).

각 단계는 독립 가치가 있어, 1단계만 해도 즉시 효용.

---

## 8. 리스크·고려사항

- **무료 API 한도**: DART/FRED/AlphaVantage 모두 호출 제한 → 캐싱·배치 필요.
- **뉴스 노이즈/신뢰도**: 헤드라인만으론 오판 위험 → LLM 분류 + 출처 표기, 점수 반영은 보수적으로(4단계 신중).
- **시차**: 미국장은 한국 새벽 → 수집 시점 설계 중요(06:30 브리프).
- **자동매매 아님**: 이슈는 **판단 보조 정보**일 뿐, `kis_trader`는 여전히 명시적 주문만.
- **데이터 책임 분리**: 조사팀은 "수집·정규화"만, 매수/매도 판단은 소미, 의사결정은 예원·사람.

## 9. 결정이 필요한 항목

1. 에이전트 이름(가칭 행크·유나·레온 확정 여부)과 폴더 구조(지역별 3폴더 vs 단일 스킬).
2. 1단계 착수 범위(한국 코어부터 vs 미국 거시 병행).
3. 신규 발급할 키(DART·ECOS·FRED 등) — 보유분이 있으면 `.env`에 추가.
4. 점수 반영(4단계)까지 갈지, 당분간 "브리프 제공"까지만 둘지.
