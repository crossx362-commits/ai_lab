# 에이전트 교차검증 시스템 설계

**날짜:** 2026-06-11  
**프로젝트:** ai-team (VS Code Extension)  
**상태:** 승인됨

---

## 목표

모든 에이전트 출력이 맹목적으로 확정되지 않고, 중요한 결정일수록 관련 전문 에이전트들의 병렬 의견 수렴을 거쳐 예원(CEO)이 최선의 결론을 도출하는 시스템.

---

## 아키텍처

```
사용자 메시지
      ↓
  예원 (CEO)
  ① 고위험 판단
      ↓ 아님 → 단독 응답
      ↓ 맞음
  ② 관련 에이전트 목록 선정
      ↓
  ③ Promise.all() 병렬 호출
  [에이전트1] [에이전트2] [에이전트3] ...
      ↓ 동시 응답 수집 (타임아웃 30s, 실패 시 스킵)
  ④ 예원 종합 → 최종 결론 출력
```

---

## 고위험 판단 기준

예원 시스템 프롬프트에 명시. 아래 중 하나 해당 시 교차검증 소집:

- 콘텐츠 발행 / 업로드 / 스케줄링
- 코드 배포 또는 외부 API 연동
- 비즈니스 / 계약 / 수익화 결정
- 보안 · 법률 관련 판단

---

## 에이전트 선정 매핑

| 도메인 | 소집 에이전트 |
|--------|--------------|
| 콘텐츠 발행 | 아린, 가희, Writer |
| 코드 / 배포 | 코다리, 경수, 케빈 |
| 비즈니스 결정 | 현빈, 로율, Researcher |
| 영상 품질 | 가희, 루나 |

도메인 매핑은 예원 프롬프트 내 정적 테이블로 관리. 추후 확장 용이.

---

## 병렬 호출 명세

### 호출 방식
- `Promise.all(agentIds.map(id => callSpecialist(id, councilPrompt)))`
- 기존 specialist 호출 함수(extension.ts 내 실제 함수명 구현 시 확인) 재사용
- 타임아웃: 30초. 실패/타임아웃 에이전트는 결과에서 제외 (에러 전파 없음)

### 각 에이전트에게 전달되는 컨텍스트
```
[원래 사용자 요청]
[태스크 도메인 및 배경]

당신의 전문 영역에서 이 결정에 대한 핵심 의견을 간결하게 제시하라.
찬성/반대/보완 의견 모두 가능. 불확실한 부분은 명시할 것.
```

### 예원의 종합 프롬프트 구조
```
[원래 사용자 요청]

--- 전문가 의견 ---
[에이전트1 이름]: [의견]
[에이전트2 이름]: [의견]
...
------------------

위 전문가 의견들을 종합해 최선의 결론을 내려라.
동의·이견·보완점을 모두 반영하고, 최종 액션 플랜을 제시하라.
```

---

## 에이전트 자기요청 (Self-Request)

각 에이전트 시스템 프롬프트에 다음 규칙 추가:

> "판단이 불확실하거나 다른 전문 영역이 필요하다고 느끼면, 응답 맨 앞에 `[COUNCIL_NEEDED: <이유>]` 태그를 붙여라."

extension.ts 응답 파싱 시:
1. `[COUNCIL_NEEDED:...]` 태그 감지
2. 해당 에이전트 원본 응답 + 원래 태스크를 예원에게 전달
3. 예원이 고위험 판단과 동일한 교차검증 흐름 소집

---

## UI 표현

1. 회의 소집 시: 패널에 `🗣️ 교차검증 중 — [에이전트1], [에이전트2] 의견 수집` 표시
2. 수집 완료 후: 예원이 `📋 종합 결론:` 접두사와 함께 최종 답변 출력

## 구현 위치

| 파일 | 변경 내용 |
|------|----------|
| `src/extension.ts` | `runCrossValidation(task, agentIds)` 함수 추가, CEO 처리 경로에 고위험 판단 로직 삽입 |
| `src/agents.ts` | 각 `AgentDef`에 `councilDomains?: string[]` 필드 추가 (선택적) |
| CEO 시스템 프롬프트 | 고위험 기준 + 도메인-에이전트 매핑 테이블 삽입 |
| 각 에이전트 시스템 프롬프트 | `[COUNCIL_NEEDED]` 자기요청 규칙 1줄 추가 |

---

## 성공 기준

- 고위험 태스크 요청 시 예원이 자동으로 관련 에이전트를 병렬 호출
- 30초 내 응답 수집 후 종합 결론 출력
- `[COUNCIL_NEEDED]` 태그 에이전트가 발생하면 예외 없이 교차검증 소집
- 단순 질문 / 저위험 태스크는 기존 단독 응답 흐름 유지

- 검증 히스토리 영구 저장 — 별도 태스크
