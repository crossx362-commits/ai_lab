# 데일리 브리핑 (Daily Briefing — 비서)

> Anthropic `meeting-prep-agent`의 브리핑팩 패턴을 한국어·텔레그램·ai_lab 운영에 맞게 재작성. 클라이언트 미팅이 아니라 **운영자(준)를 위한 일일 브리핑**이다.

## 데이터 소스
- 에이전트 현황: `_shared/notify.py` 상태(예원·영숙 — 실존 2명만), `agent_controller.py status`
- 일정: `calendar_manager.py` / Google Calendar
- 미결: 승인요청 큐(업로드 승인 등 `upload_approval_flow.py`)

## 워크플로

### 1단계 — 수집
- **에이전트 현황**: 2명(예원·영숙) 가동/중단·최근 작업. 누락된 데몬은 보정해 표기.
- **일정**: 오늘·내일 캘린더 이벤트.
- **미결 승인요청**: 사용자 결정이 필요한 항목.

### 2단계 — 브리핑 팩 포맷
```
🗂️ [YYYY-MM-DD] 영숙 데일리 브리핑

🤖 에이전트 현황
- 예원: [가동/중단] 최근작업
- 영숙: 가동

📅 오늘 일정
- [시간] [일정]

✅ 승인 대기
- [항목] — 승인/거절 필요
```

### 3단계 — 출력
- **텔레그램**: 위 팩을 전송(`_shared/notify.send`). 핵심만, 한 장.
- 길어지는 상세(주간 리포트 등)는 `report-writer`로 노션 작성([[report-to-notion]]).

## 중요 노트
- **실존 에이전트 2명만** 보고 — 과거 가짜 에이전트 언급 금지.
- 승인요청은 **인라인**으로 명확히(노션 X — 텔레그램 인라인).
- 데이터는 실시간 상태에서 — 하드코딩 금지([[no-hardcoding]]).
