# 리포트 작성 (Report Writer — 비서)

> Anthropic `meeting-prep-agent/client-report`를 ai_lab 리포팅 규칙(텔레그램 요약 + 노션 풀)으로 재작성.

## 핵심 규칙 ([[report-to-notion]])
- **채팅/텔레그램**: 완료 통보 + 핵심 요약(3~5줄).
- **노션**: 전체 내용(표·근거·차트 설명·로그).
- 실거래 포지션 매도 승인요청은 **노션이 아니라 텔레그램 인라인**.

## 워크플로

### 1단계 — 입력 수집
예원 스킬 산출물 또는 사용자 지정 내용을 받는다. 모든 수치 출처 동반 확인.

### 2단계 — 리포트 구조
```
# [제목] — [YYYY-MM-DD]

## 요약 (TL;DR)
- 결론 3~5줄

## 본문
- 섹션별 표/근거/수급/밸류

## 액션 & 리스크
- 제안 액션(모의=자동/실거래=승인)
- 무효화 조건·리스크

## 출처
- KIS/공시/데이터 출처, 추정 항목 [추정] 표기
```

### 3단계 — 발행
- 노션 페이지 생성/갱신(Notion MCP 또는 `reports_manager.py`).
- 텔레그램엔 제목 + TL;DR + 노션 링크만 전송.

## 중요 노트
- 노션에 풀, 채팅엔 요약 — 절대 채팅에 전체 붙여넣지 않는다.
- 승인이 필요한 항목은 텔레그램 인라인으로 분리.
- 출처 없는 수치 금지, 하드코딩 금지([[no-hardcoding]]).
