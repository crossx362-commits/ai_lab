## 3대 제약 (동시 적용)

### 1. "모르면 모른다"
신뢰할 출처가 없으면 그렇게 말한다. 추측·유추로 채우지 않는다.
"해당 데이터 없음"은 항상 유효한 답이다. (예: 경쟁사 최신 기능 출시일 미확인 → "확인 안 됨"으로 표기, 지어내지 않음)

### 2. 모든 주장에 출처
- 산출물 주장 = `output/cache/*.json` / `output/qa/petnna/*` 등 로컬 산출물 인용
- 뉴스·트렌드 주장 = `research.web_brief` 결과의 URL·매체명 인용
- 저장소 관련 주장 = 파일 경로 + 줄번호
- 출처를 못 찾은 주장은 리포트에서 제외 (철회)

### 3. 원문 우선
뉴스 기반 판단 시 LLM 요약이 아니라 원문 문장을 먼저 발췌하고, 그 인용을 근거로 분석한다.

## 출처 캐스케이드 (비용 순서 강제)

1. **로컬 캐시/로그 (비용 0)**: `output/cache/`, `output/bot_logs/`, `output/qa/petnna/` — 기존 산출물 근거
2. **웹 검색 스니펫 (저비용)**: 스니펫 인용으로 충분하면 여기서 정지
3. **전문 페치 (고비용)**: 스니펫이 모호하거나 원문 인용 필요 시만

## 토큰 예산 (AI_토큰_최적화_스킬과 정렬)

- 질문/리포트당 웹검색 최대 5회, 전문 페치 최대 3회
- 한도 도달 시: 확인분 요약 + 미확인 목록 명시 + 추가 조사 여부 질문
- 병렬 조회 OK, 직렬 재시도 루프 금지

## 인정되는 인용

| 인정 | 불인정 |
|---|---|
| 캐시 파일명 + 타임스탬프 | "일반적으로 알려진 바" |
| 뉴스 URL + 매체명 | 모델 기억 ("~로 알고 있음"은 출처불가 명시 시만 허용) |
| 로컬 산출물 필드 값 | 근거 없는 추정치 |

## 이 모드가 아닌 것

- 기본값 아님 — 아이디어 발굴·전략 브레인스토밍엔 강제하지 않음
- 느리게 하라는 뜻 아님 — 병렬로 효율적으로
- 종합 금지 아님 — 근거 기반 입력이면 새로운 결론 도출 가능
