# Skill Creator

에이전트의 SKILL.md를 새로 만들거나 기존 것을 개선하는 절차.

## 이 프로젝트의 스킬 구조

```
ai-team/
└── .agent/skills/
    └── <에이전트명>/
        └── SKILL.md        ← 에이전트당 하나
└── _shared/
    ├── 공통_스킬_지식.md
    ├── 멀티에이전트_토론_스킬.md
    └── skill-creator.md    ← 이 파일
```

**SKILL.md 필수 구조:**
```markdown
---
name: <에이전트 식별자>
description: <트리거 조건 + 역할 한 줄 설명. 구체적이고 약간 "pushy"하게>
---

# 에이전트명 — 역할

## Section 1. Persona
## Section 2. 핵심 미션
## Section 3. 제약·금지 규칙
```

## 스킬 제작 흐름

### 1단계: 의도 파악

사용자에게 확인할 것:
- 이 스킬이 무엇을 할 수 있어야 하는가?
- 언제 트리거되어야 하는가? (어떤 키워드·상황)
- 결과물의 형태는?
- 어느 에이전트에 추가할 것인가?

기존 대화에서 이미 답이 있으면 추출해서 확인만 받는다.

### 2단계: SKILL.md 초안 작성

**좋은 description 작성법:**
- 역할 + 트리거 조건을 함께 명시
- 언더트리거를 막기 위해 약간 능동적으로 작성
- 나쁜 예: `"데이터를 분석한다"`
- 좋은 예: `"월간 수익·KPI 분석 전문가. 수익, 매출, 분석, 보고서 키워드가 나오면 즉시 활성화"`

**본문 작성 원칙:**
- 명령형 사용 ("분석한다" 아닌 "분석하라")
- MUST/NEVER 대신 이유 설명 ("중복 방지를 위해 ~한다")
- 500줄 이내 유지
- 대형 참조 파일은 `_shared/` 에 두고 링크

### 3단계: 직접 테스트 (서브에이전트 없이)

테스트 프롬프트 2~3개 작성 후, 해당 SKILL.md를 읽고 직접 그 에이전트처럼 실행해본다.

테스트 기록 양식:
```
## 테스트 케이스 1
**프롬프트**: (사용자가 실제로 입력할 내용)
**예상 결과**: (무엇이 나와야 하는가)
**실행 결과**: (직접 실행한 결과 요약)
**판정**: ✅ 통과 / ❌ 실패 — 이유
```

### 4단계: 피드백 → 개선 반복

개선 시 주의:
- 테스트 케이스에만 맞는 과적합 수정 지양
- 왜 그렇게 해야 하는지 이유를 스킬 본문에 녹여라
- 여러 테스트에서 동일한 패턴이 나오면 본문에 일반화해서 반영
- 불필요한 지시는 제거 (가볍고 이유가 명확한 스킬이 더 잘 작동)

### 5단계: 적용

완성된 스킬은 해당 에이전트의 SKILL.md에 반영:
- **새 섹션 추가**: 기존 파일 끝에 `---` 구분선 후 붙이기
- **기존 섹션 교체**: 해당 섹션만 수정, 나머지 보존
- **공유 스킬**: `_shared/` 에 저장 후 각 에이전트에 참조 섹션 추가

## description 최적화 팁

에이전트가 스킬을 잘 트리거하려면 description이 결정적이다.

나쁜 description:
```
데이터 분석 및 보고서 작성 전문가
```

좋은 description:
```
월간·주간 수익·KPI 분석 전문가. 수익, 매출, 전환율, 분석, 리포트, 보고서, 
실적 키워드가 등장하거나 비즈니스 성과 검토를 요청할 때 즉시 활성화한다.
```

description 개선 후에는 반드시 트리거 여부를 테스트 프롬프트로 확인한다.

## 체크리스트

스킬 완성 전 확인:
- [ ] description에 트리거 조건이 명확하게 있는가
- [ ] 스킬 본문이 500줄 이내인가
- [ ] 모든 참조 파일(`_shared/`)이 실제로 존재하는가
- [ ] 테스트 2~3개 모두 통과했는가
- [ ] 다른 에이전트 역할과 충돌하지 않는가
- [ ] YAML frontmatter (name, description) 정확한가
