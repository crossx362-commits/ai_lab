# 서버리스

## 핵심 개념

* **FaaS (Function as a Service):** 이벤트 기반으로 실행되는 함수 단위 코드. 서버 관리 불필요.
* **BaaS (Backend as a Service):** 인증, 데이터베이스 등 백엔드 기능 제공. 직접 구축 없이 사용.
* **이벤트 기반 아키텍처:** 특정 이벤트(HTTP 요청, 메시지, 파일 업로드) 발생 시 함수 실행.
* **자동 스케일링:** 트래픽에 따라 자동으로 리소스 확장/축소. 비용 효율적.
* **Stateful vs. Stateless:**  Stateless 함수는 이전 요청 상태를 유지하지 않음.

## 실전 적용

* **API Gateway + Lambda (AWS):** REST API 구축. Lambda 함수는 API 요청 처리.
* **Cloud Functions (GCP):** 이미지 썸네일 생성, 데이터 변환 등.
* **Firebase Functions:** 모바일 앱 백엔드 구축. 인증, 데이터베이스 연동 용이.
* **Serverless Framework:** 다양한 클라우드 플랫폼에 배포 자동화.
