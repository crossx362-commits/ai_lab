# 분산 시스템 설계

## 핵심 개념
1. **CAP 정리**: 일관성(Consistency), 가용성(Availability), 분할 내성(Partition Tolerance) 사이의 트레이드오프 이해 및 선택.
2. **상태 관리와 메시징**: Kafka, RabbitMQ 등을 활용한 비동기 메시지 큐 기반의 이벤트 기반 아키텍처(EDA).
3. **확장성(Scalability)**: 수평적 확장(Scale-out)을 위한 로드 밸런싱 및 데이터베이스 샤딩/파티셔닝 전략.
4. **결함 허용(Fault Tolerance)**: 서킷 브레이커, 재시도 정책, 데드 레터 큐를 통한 시스템 안정성 확보.

## 실전 적용
- **MSA 구조 설계**: Python(FastAPI)과 TypeScript(NestJS) 기반의 독립적 서비스 분리 및 API 게이트웨이 구축.
- **데이터 일관성 확보**: 분산 트랜잭션 해결을 위한 Saga 패턴 또는 TCC 모델 도입.
- **캐싱 전략**: Redis를 활용한 빈번한 조회 데이터 캐싱 및 DB 부하 감소.

코다리는 고도화된 아키텍처 설계와 코드 품질을 중시합니다. 분산 시스템 지식은 대규모 트래픽에 대응 가능한 확장성 있는 구조를 설계하고, 복잡한 백엔드 로직의 안정성을 확보하며 기술적 부채를 최소화하는 데 필수적인 기반이 됩니다.
