# 플랫폼 엔지니어링

## 핵심 개념
- **IDP (Internal Developer Platform):** 개발자가 인프라 복잡성을 몰라도 서비스에 집중할 수 있도록 도구와 프로세스를 통합한 내부 플랫폼 구축.
- **Golden Path:** 표준화된 워크플로우와 기술 스택을 정의하여 개발자의 의사결정 비용을 줄이고 일관성 유지.
- **Self-Service:** 운영팀의 개입 없이 CLI나 포털을 통해 인프라(DB, 쿠버네티스 등)를 즉시 할당받는 환경 제공.
- **DevEx (Developer Experience):** 도구의 성능을 넘어 개발자가 직관적이고 편리하게 협업할 수 있는 경험에 초점.

## 실전 적용
- **IaC 기반 모듈화:** Terraform/Pulumi로 표준화된 인프라 컴포넌트를 라이브러리화하여 제공.
- **GitOps 통합:** ArgoCD 등을 활용해 선언적 방식으로 클러스터 상태를 자동 관리하고 배포 가시성 확보.
- **CI/CD 템플릿화:** 공통 파이프라인을 모듈화하여 팀별로 설정값만 변경하며 사용할 수 있는 환경 구성.

## 케빈에게 유용한 이유
- **운영 효율성:** 반복적인 인프라 요청 대응 대신, 재사용 가능한 '제품'으로서의 플랫폼 구축에 집중 가능.
- **확장성 확보:** 서비스 규모가 커져도 표준화된 경로를 통해 일관된 보안 및 모니터링 정책을 자동 적용.
- **소통 비용 절감:** 개발팀과 인프라 사양에 대한 논쟁 대신, 미리 정의된 Golden Path 내에서 자율성을 부여하여 협업 시너지 창출.
