# YouTube Music Optimization Skill

## 목적

YouTube 음악 콘텐츠의 제목, 음악 생성 프롬프트, Description, SEO 키워드, 해시태그, 업로드 시간을 최적화하여 검색 노출과 추천 알고리즘 적합도를 높인다.

---

# 1. 제목 생성 규칙

매 실행 시 전날 미국 YouTube 인기 음악 영상 상위 100개 제목을 수집하고 분석한다.

분석 대상:

* 제목 구조
* 제목 길이
* 키워드 배치
* 특수문자 사용
* 아티스트 표기 방식
* Official Video 표기 방식
* Featuring 표기 방식

분석 결과를 바탕으로 새로운 제목을 생성한다.

## 원칙

* 고정 제목 공식 사용 금지
* 매 곡마다 새로운 제목 생성
* 제목의 분위기는 음악 내용과 일치해야 함
* 실제 음악의 감정과 무관한 키워드 삽입 금지
* 최근 트렌드 제목 패턴 적극 반영

## 금지 키워드

* Lofi
* Lo-fi
* Study Beats
* Chill Beats
* Sleep Music
* White Noise
* Ambient Study

## 폴백

분석 실패 시:

[KEYWORD] [Official Music Video] (감성 설명)

생성 패턴은 knowledge/title_patterns.json에 저장한다.

---

# 2. 음악 프롬프트 생성 규칙

모든 음악 프롬프트는 아래 구조를 따른다.

[장르/시대]
+
[템포/무드]
+
[악기 구성]
+
[보컬 스타일]
+
[주제/가사]

## 장르 우선순위

1. Japanese City Pop × K-Pop Fusion
2. Emotional Hip-Hop × R&B × Pop
3. 기타 장르

## 에너지 기준

권장:

* City Pop 110 BPM 이상
* K-Pop Dance 120 BPM 이상
* Hip-Hop 110 BPM 이상
* R&B 90 BPM 이상

## 분위기 방향

우선:

* 에너제틱
* 희망적
* 자신감 있는
* 몰입감 있는
* 대중적인

지양:

* 수면 유도
* 지나치게 정적인 분위기
* 백색소음 성향
* 공부용 BGM 성향

## 생성 원칙

매 생성 시 아래 요소를 분석한다.

* 제목
* 키워드
* 트렌드
* 장르
* 분위기

고정 프롬프트 재사용 금지.

모든 곡은 새로운 프롬프트로 생성한다.

---

# 3. 메타데이터 생성 규칙

Description, SEO 키워드, 해시태그, 분위기 설명, 추천 상황은 모두 실제 음악 내용을 기반으로 생성한다.

고정 문구 사용 금지.

## 분석 대상

* 제목
* 음악 프롬프트
* 장르
* BPM
* 악기
* 보컬 스타일
* 주제
* 분위기
* 가사 키워드

---

## Description 생성

음악의 실제 감정과 장면을 자연스럽게 설명한다.

설명은 음악이 전달하는 경험을 표현해야 한다.

반복 문구 사용 금지.

---

## 필수 메타데이터

Description 내부에 포함:

🎹 Genre / Era

🎸 Instruments

🎙 Vocal Style

✨ Theme

---

## SEO 키워드 생성

다음 요소에서 추출:

* 장르
* 악기
* 분위기
* 주제
* 상황
* 문화적 키워드

검색량이 높고 음악과 관련성이 높은 키워드를 우선 사용한다.

---

## 해시태그 생성

음악 특성에 맞게 동적으로 생성한다.

권장:

* 3~10개

예시 범주:

* 장르
* 분위기
* 테마
* 상황
* 아티스트명

고정 해시태그 사용 금지.

---

## 분위기 설명 생성

음악이 전달하는 감정을 구체적으로 표현한다.

예:

* 네온 도시의 에너지
* 여름 해변의 자유로움
* 청춘의 자신감
* 미래를 향한 질주
* 깊은 감성의 회상

음악과 무관한 감정 묘사 금지.

---

## 추천 상황 생성

실제 음악 분위기에 맞게 생성한다.

예:

* 출근길
* 운동
* 드라이브
* 여행
* 카페 작업
* 홈파티
* 야간 산책
* 주말 휴식

고정 추천 상황 사용 금지.

---

# 4. 반복 콘텐츠 방지 규칙

아래 표현을 반복적으로 사용하는 것을 금지한다.

* 네온 아래
* 감성 충전
* 늦은 밤 드라이브
* 벚꽃 흩날리는 거리
* 여름 바닷가
* 조용한 밤
* 편안한 휴식

동일 문구 반복 시 새로운 표현으로 대체한다.

항상 음악의 실제 특성을 우선 반영한다.

# 6. 최종 원칙

모든 생성 결과는 다음 순서를 따른다.

음악 내용 분석
→ 제목 생성
→ 음악 프롬프트 생성
→ Description 생성
→ SEO 키워드 생성
→ 해시태그 생성
→ 추천 상황 생성
→ 예약 업로드 설정

제목, Description, SEO, 해시태그, 분위기 설명, 추천 상황은 항상 실제 음악 내용을 기반으로 생성하며 고정 템플릿 반복을 금지한다.
