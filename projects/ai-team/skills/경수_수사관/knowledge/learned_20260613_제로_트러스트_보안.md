# 제로 트러스트 보안

## 핵심 개념
1. **Never Trust, Always Verify**: 모든 접속 요청을 신뢰하지 않으며, 매 순간 인증과 권한 검증 수행.
2. **최소 권한 원칙 (PoLP)**: 사용자 및 기기에 업무 수행에 필요한 최소한의 권한만 부여.
3. **마이크로 세그멘테이션**: 네트워크를 미세하게 분할하여 침해 사고 발생 시 피해 확산(Lateral Movement)을 차단.
4. **컨텍스트 기반 접근 제어**: 위치, 기기 상태, 사용자 행동 등 실시간 상황 정보를 바탕으로 권한 부여.

## 실전 적용
- **MFA 및 조건부 액세스**: 단순 비밀번호를 넘어 MFA를 필수화하고, 승인된 기기/지역에서만 접근 허용.
- **IAM 기반 세분화**: 역할(RBAC)을 넘어 속성(ABAC) 기반의 정교한 권한 관리 수행.
- **지속적 모니터링**: 세션 유지 중에도 비정상적인 행동 패턴이 감지되면 즉시 인증 재요구 또는 차단.

## 경수에게 유용한 이유
경수의 주요 업무인 **이상행동 모니터링**과 **리스크 탐지**에서 강력한 기반이 됩니다. 기존의 경계 보안 방식보다 훨씬 세밀한 로그를 수집하게 되므로, 계정 탈취나 내부자 위협과 같은 정교한 공격을 실시간 컨텍스트 분석을 통해 신속하게 식별할 수 있습니다.
