# 제로 트러스트 보안

## 핵심 개념
1. **Never Trust, Always Verify**: 모든 접속 요청을 신뢰하지 않으며, 매 단계에서 인증과 권한 검증 수행.
2. **최소 권한 원칙 (PoLP)**: 사용자 및 기기에 업무 수행에 필요한 최소한의 권한만 부여.
3. **마이크로 세그멘테이션**: 네트워크를 잘게 쪼개어 공격자의 횡적 이동(Lateral Movement)을 차단.
4. **동적 정책 적용**: 위치, 기기 상태, 시간 등 컨텍스트를 기반으로 실시간 접근 제어.

## 실전 적용
- **ZTNA 도입**: 기존 VPN 대신 사용자 식별 중심의 Zero Trust Network Access로 전환.
- **MFA 강제화**: 모든 중요 시스템 접근 시 다요소 인증 필수 적용.
- **IAM 통합**: Identity 및 Access Management를 통한 정밀한 권한 매핑.
- **지속적 모니터링**: 세션 유지 중에도 비정상 행위 감지 시 즉각적인 차단 조치.

## 경수에게 유용한 이유
- **이상행동 모니터링 강화**: 사용자별 상세 로그가 생성되어 정교한 이상 패턴 분석 가능.
- **리스크 탐지 및 격리**: 침해 사고 발생 시 마이크로 세그멘테이션을 통해 피해 범위(Blast Radius)를 최소화하고 신속하게 격리 가능.
- **정밀 보안 대응**: 경계 기반 방어의 한계를 극복하여 개별 리소스 수준에서 실시간 위협 탐지 및 대응 가능.
