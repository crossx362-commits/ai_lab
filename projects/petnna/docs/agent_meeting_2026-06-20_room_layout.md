# 펫과나 마이펫 방 레이아웃 에이전트 회의록

일시: 2026-06-20 10:25 KST  
주제: 마이펫 방의 거실형 레이아웃, 집사-펫 연결선, 사용자 선택형 방 구조 고도화  
참석: 예원(총괄), 영숙(운영), 코다리(개발), 케빈(인프라), 티모(디자인), 로율(법무), 경수(보안), 시그널(사용 흐름 분석), 펄스(전략), 데이브(보수적 리스크), 레오(성장 실험), 아린(온보딩), 루나(콘텐츠)

## 회의 목적

마이펫 방은 펫과나의 첫 감정 접점이다. 단순히 펫 목록을 보여주는 화면이 아니라, 사용자가 "내 펫과 내가 함께 사는 방"이라고 느끼게 해야 한다. 이번 회의는 최근 반영된 거실형/교감형 선택 구조를 기준으로, 다음 개선 방향과 우선순위를 정하는 데 집중했다.

## 현재 확인된 상태

- 마이펫 방은 기본값이 `거실형`으로 바뀌었다.
- 방 설정에서 `거실형`과 `교감형`을 선택할 수 있다.
- 집사와 펫 사이 연결선은 점선 목줄 느낌에서 SVG 교감 라인으로 개선되었다.
- 모바일/데스크톱 1~12마리 배치에서 펫끼리 겹침과 집사 영역 침범은 테스트로 확인했다.
- 아직 사용자는 레이아웃을 고르기 전에 실제 미리보기를 보지 못한다.
- 거실형 배경은 은은한 소파/러그 느낌만 있으며, 테마 선택이나 가구 꾸미기는 아직 없다.

## 에이전트별 발언 요약

### 예원 - 제품 총괄

마이펫 방은 "기능 화면"이 아니라 제품의 정체성을 보여주는 홈이다. 현재 거실형 기본값은 방향이 맞다. 다만 레이아웃 선택이 설정 안에만 있으면 사용자가 기능을 발견하지 못할 수 있다.

결정: 첫 진입 후 한 번은 "방 분위기 골라보기"를 자연스럽게 노출한다.

### 티모 - UI/UX 디자인

거실형은 감성적으로 맞지만, 선택 버튼만으로는 사용자가 차이를 이해하기 어렵다. 카드형 미리보기 2개가 필요하다. "거실형"은 소파/러그 중심, "교감형"은 관계 라인 중심으로 썸네일을 다르게 보여줘야 한다.

결정: 방 설정 안에 작은 미리보기 카드와 선택 상태를 추가한다.

### 코다리 - 개발

현재 구현은 펫 객체의 `roomLayout`에 저장되는 구조라 가볍고 유지보수 가능하다. 다음 단계에서 자유 배치 드래그까지 바로 가면 모바일 좌표 저장, 충돌 보정, 복구 로직이 커진다. 프리셋을 먼저 늘리는 편이 안정적이다.

결정: 자유 배치는 보류하고 `거실형`, `교감형`, `창가형`, `놀이매트형` 같은 프리셋 확장으로 간다.

### 영숙 - 운영

사용자 문의 관점에서는 "방 설정이 어디 있나요?", "선택했는데 저장됐나요?"가 생길 수 있다. 선택 즉시 토스트가 뜨는 것은 좋다. 여기에 현재 방 이름 아래에 작은 레이아웃 배지를 보여주면 사용자가 안심한다.

결정: 방 제목 근처에 현재 레이아웃 배지를 추가한다.

### 케빈 - 인프라

레이아웃 선택은 로컬 저장만으로도 동작하지만, 로그인 사용자는 Supabase 동기화가 필요하다. 이미 `updatePetInSupabase(current)` 호출이 붙어 있으니 방향은 맞다. 다만 DB 스키마에 `roomLayout` 저장 허용 여부를 점검해야 한다.

결정: Supabase `pets` 저장 필드에 `roomLayout`이 누락되지 않는지 확인한다.

### 로율 - 법무

방 꾸미기 기능이 유료화되면 "구매한 아이템의 소유", "환불", "디지털 콘텐츠 제공" 문구가 필요하다. 지금처럼 무료 레이아웃 프리셋 선택은 법적 리스크가 낮다. 유료 가구/테마로 확장할 때 약관 문구를 먼저 정리해야 한다.

결정: 현재 무료 프리셋은 문제 없음. 유료 꾸미기 전 약관 검토 필요.

### 경수 - 보안

사용자 업로드 사진과 방 설정은 사용자 데이터다. 방 꾸미기 코드가 HTML 문자열로 커질수록 이미지 URL이나 이름 렌더링에서 XSS 가능성을 봐야 한다. 레이아웃 ID는 허용 목록으로 제한한 현재 방식이 안전하다.

결정: 레이아웃 ID는 계속 allowlist 방식으로만 처리한다.

### 시그널 - 사용 흐름 분석

마이펫 방에서 사용자가 오래 머무는 이유는 "바꿨을 때 바로 보이는 피드백"이다. 레이아웃 선택 직후 방이 즉시 바뀌는 현재 흐름은 좋다. 다음은 펫 상태나 날씨에 따라 방 분위기가 미세하게 달라지는 피드백이다.

결정: 날씨/시간대 기반 방 조명 변화를 후보로 둔다.

### 펄스 - 전략

방 꾸미기는 retention과 premium 전환에 유리하다. 하지만 초반부터 유료 벽을 치면 감정 몰입을 끊는다. 무료 프리셋 2~4개를 충분히 예쁘게 제공하고, 프리미엄은 시즌 테마나 월간 리포트와 연결하는 편이 낫다.

결정: 기본 레이아웃은 무료, 시즌 테마는 추후 프리미엄 후보로 둔다.

### 데이브 - 보수적 리스크

현재 우선순위는 예쁜 새 기능보다 깨지지 않는 방이다. 펫 수가 많아져도 겹치지 않는 검증이 있으니 좋다. 다음 변경도 반드시 1~12마리 모바일/데스크톱 좌표 검증을 유지해야 한다.

결정: 레이아웃 추가 시 좌표 충돌 테스트를 필수로 유지한다.

### 레오 - 성장 실험

방 선택은 공유 카드와 연결하면 바이럴 포인트가 된다. "우리 집 거실 공개" 같은 캡션으로 소셜 피드에 공유할 수 있으면 기능이 살아난다. 단, 먼저 방 자체가 예뻐야 한다.

결정: 방 공유 카드는 2차 실험으로 둔다.

### 아린 - 온보딩

처음 펫 등록 후 바로 방 설정을 보여주면 사용자가 앱에 애착을 느낀다. 긴 튜토리얼보다 "초코 방 분위기를 골라볼까요?" 같은 한 문장과 두 개 카드가 좋다.

결정: 첫 펫 등록 직후 방 레이아웃 선택 제안을 추가 후보로 둔다.

### 루나 - 콘텐츠

방은 말풍선과 연결되어야 살아난다. 거실형이면 "오늘은 소파 옆에서 쉬고 싶어요", 교감형이면 "집사랑 마음이 반짝 이어졌어요"처럼 레이아웃별 대사를 다르게 줄 수 있다.

결정: 레이아웃별 펫 말풍선 카피를 만든다.

### 티모+루나 - GPT 이미지 생성 활용

현재 거실형 배경은 CSS 장식만으로 분위기를 만들고 있어 가볍고 안정적이지만, 실제 "방"처럼 기억되는 감성은 아직 약하다. GPT 이미지 생성을 활용해 소파, 러그, 창가 햇살, 쿠션, 낮잠 담요 같은 배경 오브젝트를 반려동물 앱 톤에 맞게 제작하면 화면의 생동감이 크게 올라간다.

이미지는 전체 배경을 한 장으로 깔기보다, 투명 PNG 오브젝트 세트로 나누는 편이 좋다. 예를 들어 `cozy sofa`, `round rug`, `sunny window`, `pet cushion`, `toy basket`을 각각 생성해 CSS로 배치하면 모바일 반응형 대응과 테마 교체가 쉽다.

결정: GPT 이미지 생성은 `거실형` 고도화의 1차 시각 에셋 제작 수단으로 사용한다.

## 합의된 방향

1. `거실형` 기본값은 유지한다.
2. `교감형`은 관계 라인이 잘 보이는 보조 프리셋으로 유지한다.
3. 다음 개선은 새 레이아웃 대량 추가보다 "선택 경험"을 먼저 고도화한다.
4. 레이아웃 선택은 즉시 저장되고, Supabase 동기화도 유지한다.
5. 자유 드래그 배치는 아직 하지 않는다.
6. 모든 레이아웃은 모바일/데스크톱 1~12마리 겹침 테스트를 통과해야 한다.

## 다음 실행 우선순위

### P0 - 선택 경험 완성

1. 방 설정에 레이아웃 미리보기 카드를 추가한다.
2. 방 제목 근처에 현재 레이아웃 배지를 표시한다.
3. 선택 버튼 문구를 "거실형", "교감형"에서 한 줄 설명이 있는 카드로 바꾼다.

### P1 - 거실형 감성 강화

1. 거실형 배경에 소파/러그/창 조명 느낌을 조금 더 명확히 준다.
2. 집사와 펫 위치가 "바닥에 앉아 있는" 느낌으로 보이도록 하단 안정감을 강화한다.
3. 레이아웃별 말풍선 문구를 분리한다.
4. GPT 이미지 생성으로 투명 배경 거실 오브젝트 세트를 제작한다.

### P1.5 - GPT 이미지 생성 에셋 후보

1. `cozy-sofa.png`: 따뜻한 크림/살구 톤의 낮은 소파, 펫 방해 없이 뒤쪽 배경에 깔리는 형태.
2. `soft-round-rug.png`: 펫과 집사가 앉는 영역을 묶어주는 둥근 러그.
3. `sunny-window.png`: 오전/낮 시간대 방 분위기를 살리는 창가 햇살 오브젝트.
4. `pet-cushion-set.png`: 강아지/고양이/토끼 모두 어울리는 작은 쿠션 세트.
5. `toy-basket.png`: 장난감 바구니와 공, 뼈다귀, 실타래 등 방을 살아 보이게 하는 소품.

이미지 생성 원칙:

- 투명 배경 PNG로 만든다.
- 펫/집사 아바타보다 시선이 강하지 않게 채도를 낮춘다.
- 너무 사실적인 사진보다 부드러운 3D/일러스트 중간 질감으로 맞춘다.
- 모바일에서 잘려도 어색하지 않게 오브젝트 중심부를 단순하게 둔다.
- 저작권 리스크가 있는 특정 캐릭터, 브랜드 가구, 유명 IP 스타일은 피한다.

### P2 - 프리셋 확장

1. `창가형`: 조용한 휴식, 고양이/토끼에 어울림.
2. `놀이매트형`: 장난감/활동적인 펫에 어울림.
3. `캠핑형`: 산책/야외 기록과 연결 가능.

### P3 - 공유와 수익화

1. 방 스냅샷 공유 카드 후보를 만든다.
2. 시즌 테마를 프리미엄 후보로 검토한다.
3. 유료 테마 전에 약관/환불/디지털 콘텐츠 문구를 점검한다.

## 최종 결정

다음 작업은 `레이아웃 미리보기 카드 + 현재 레이아웃 배지`가 1순위다. 지금의 거실형 구현은 방향이 맞으므로 갈아엎지 않고, 사용자가 더 쉽게 발견하고 선택 결과를 더 확실히 느끼게 만든다.

그다음 단계로 GPT 이미지 생성을 활용해 거실형 전용 투명 PNG 오브젝트를 만들고, CSS 배경 장식과 섞어 방을 더 생생하게 만든다. 첫 에셋은 소파, 러그, 창가 햇살, 펫 쿠션, 장난감 바구니 5종으로 시작한다.
