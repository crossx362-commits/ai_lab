# 🗓️ 영숙 스케줄 중앙 관리 시스템
**구축일**: 2026-06-02  
**담당**: 영숙 비서  
**목적**: 모든 에이전트 스케줄을 영숙이 중앙 관리, CEO 보고 후 에이전트 지시

---

## 📊 시스템 개요

### 기존 방식의 문제점
- 각 에이전트가 개별적으로 스케줄 실행
- 스케줄 충돌 및 우선순위 조정 불가
- CEO가 전체 스케줄 현황 파악 어려움
- 에이전트 간 협업 스케줄 조율 불가

### 새로운 방식 (중앙 관리)
```
1. 영숙이 모든 스케줄 모니터링 (60초 주기)
2. 스케줄 시간 도래 시 → CEO 예원에게 보고
3. CEO 승인 (자동) → 해당 에이전트에게 작업 지시
4. 작업 완료 후 → 사장님께 최종 보고
```

## 🎯 주요 기능

### 1. 스케줄 JSON 관리
**파일**: `skills/영숙_비서/tools/schedules.json`

```json
{
  "version": "1.0.0",
  "schedules": [
    {
      "id": "gahee_morning_inspection",
      "agent": "가희",
      "task": "아침 콘텐츠 검수",
      "cron": "0 7 * * *",
      "description": "YouTube + Instagram 검수",
      "command": "가희 검수 돌려",
      "enabled": true,
      "priority": "high"
    }
  ]
}
```

**필드 설명**:
- `id`: 스케줄 고유 ID
- `agent`: 담당 에이전트 이름
- `task`: 작업 설명
- `cron`: Cron 표현식 (분 시 일 월 요일)
- `description`: 상세 설명
- `command`: 실행 명령어 (영숙이 텔레그램 형식)
- `enabled`: 활성화 여부
- `priority`: 우선순위 (critical, high, medium, low)

### 2. 스케줄 매니저
**파일**: `skills/영숙_비서/tools/schedule_manager.py`

#### 주요 함수
- `load_schedules()`: JSON에서 스케줄 로드
- `should_run()`: croniter로 실행 시간 판단
- `execute_schedule()`: CEO 보고 → 에이전트 지시
- `schedule_loop()`: 백그라운드 데몬 실행
- `list_schedules()`: 스케줄 목록 출력

## 🛠️ 트러블슈팅

### 문제: 스케줄이 실행되지 않음
**원인 1**: 스케줄러가 실행 중이 아님
```bash
# 프로세스 확인
Get-Process pythonw | Where-Object { 
    (Get-WmiObject Win32_Process -Filter "ProcessId = $($_.Id)").CommandLine -match "schedule_manager" 
}
```

**원인 2**: enabled: false로 설정됨
```json
{
  "enabled": true  // false로 되어 있으면 실행 안 됨
}
```

**원인 3**: Cron 표현식 오류
```bash
# Cron 표현식 테스트
python -c "from croniter import croniter; print(croniter('0 9 * * *').get_next())"
```

### 문제: CEO Dispatcher 호출 실패
**원인**: yewon_dispatcher import 오류

**해결**:
```python
# schedule_manager.py Line 20-22
```

## 📊 성능 및 리소스

### 메모리 사용량
- 스케줄러: ~50MB
- 스케줄 1개당: ~1KB (JSON)
- 총 20개 스케줄: ~50MB + 20KB

### CPU 사용률
- Idle: ~0.1% (60초 sleep)
- 스케줄 체크: ~0.5% (1초 미만)
- 에이전트 실행: 에이전트별 상이

### 네트워크
- 텔레그램 알림: 스케줄당 2회 (보고 + 결과)
- 예상 트래픽: 하루 40회 × 1KB = 40KB
