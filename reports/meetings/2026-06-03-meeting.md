## 경수 (보안)

### 보안 취약점 발견 (실제 코드 검토)

**취약점 1: API 키 클라이언트 노출 (index.html:54-60)**
```javascript
window._env_ = {
    "SUPABASE_URL": "https://nlgjsdffgkygaylbjooc.supabase.co",
    "SUPABASE_ANON_KEY": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "GEMINI_API_KEY": "",
    "STRIPE_PAYMENT_LINK": ""
};
```
- **Supabase anon key가 HTML에 평문 노출** — anon key 자체는 설계상 공개 가능하나, RLS(Row Level Security) 정책이 정확히 설정되지 않으면 전체 DB 읽기/쓰기 가능
- GEMINI_API_KEY가 활성화되면 동일 방식으로 노출 → Gemini API 요금 도용 위험

**취약점 2: 프리미엄 상태 localStorage 단독 의존 (freemium.js:12-14)**
```javascript
function isPremium() {
    return !!localStorage.getItem("petna_premium");
}
```
- `localStorage.setItem("petna_premium", "anything")` 한 줄로 프리미엄 무력화 가능
- 서버 검증 없음 → 결제 우회 100% 가능

**취약점 3: XSS 잠재 위험 (social.js:32-44)**
- `escapeHtml()` 함수가 친구 닉네임에 적용되나, `f.personality`, `f.petBreed` 등 일부 필드는 `escapeHtml()` 누락 가능성 확인 필요

**취약점 4: activatePremiumDemo() 프로덕션 노출 (freemium.js:107-113)**
- 콘솔에서 `activatePremiumDemo()` 직접 호출 가능 → 결제 없이 프리미엄 활성화

### 패치 필요 항목

| 우선순위 | 취약점 | 권고 조치 |
|---------|--------|----------|
| 1 (즉시) | localStorage 프리미엄 단독 검증 | Supabase에 `user_subscriptions` 테이블 추가, 서버 검증 병행 |
| 2 (즉시) | activatePremiumDemo 노출 | 프로덕션 빌드에서 함수 제거 또는 `if(location.hostname !== 'localhost')` 가드 추가 |
| 3 (단기) | Gemini API Key 클라이언트 노출 | Supabase Edge Function으로 API 호출 프록시화 |
| 4 (단기) | Supabase RLS 정책 감사 | 모든 테이블에 `auth.uid() = user_id` 조건 확인 |
